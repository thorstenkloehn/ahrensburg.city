# Sicherheits- und Qualitätsbericht (25.03.2026, 15:26)

Dieser Bericht fasst die aktuell bekannten technischen Schwachstellen, Bugs und Verbesserungspotenziale von MeinCMS zusammen.

## 1. Bekannte Schwachstellen (Security)

### 1.1 Host-Header Spoofing Risiko
Die Mandantenerkennung basiert auf dem `Host`-Header des HTTP-Requests. Falls der vorgeschaltete Reverse Proxy (Nginx) nicht korrekt konfiguriert ist (d. h. den Header nicht validiert oder beliebige Header durchlässt), könnte ein Angreifer versuchen, auf Inhalte anderer Mandanten zuzugreifen.
- **Status:** Gemildert durch Dokumentation in `Anleitung_Start.md`.
- **Empfehlung:** Sicherstellen, dass Nginx nur bekannte Hostnamen an die App weiterleitet.

### 1.2 UserAdmin Privilegien
Das Tool `UserAdmin` hat direkten Zugriff auf die Datenbank und kann Passwörter ohne Kenntnis des alten Passworts überschreiben.
- **Status:** Akzeptiertes Risiko für Notfall-Wiederherstellung.
- **Empfehlung:** Zugriff auf das `UserAdmin` Binary auf dem Server strikt auf `root` oder den App-User beschränken.

### 1.3 Unix-Socket Berechtigungen
Der Unix-Socket unter `/run/meincms.sock` erfordert korrekte Dateisystem-Berechtigungen. Falls diese zu offen gewählt werden (`777`), könnten andere lokale Benutzer auf den Socket zugreifen.
- **Status:** Konfigurationsabhängig.
- **Empfehlung:** Socket-Verzeichnis auf den App-User und die Nginx-Gruppe (`www-data`) einschränken.

## 2. Bekannte Bugs und Compiler-Warnungen

### 2.1 Mögliche Null-Referenz in der View (`CS8602`)
In `mvc/Views/Page/Edit.cshtml` (Zeile 23) gibt es eine Warnung über eine mögliche Dereferenzierung eines Null-Verweises. Dies könnte bei fehlerhaften Artikel-Daten zu einem 500er Fehler führen.
- **Priorität:** Mittel
- **Datei:** `mvc/Views/Page/Edit.cshtml`

### 2.2 Namenskonventionen in Migrationen (`CS8981`)
Die Migrationen `thorsten` und `tt` verstoßen gegen die C#-Namenskonventionen (nur Kleinbuchstaben). Dies ist funktional unbedenklich, sollte aber bei künftigen Migrationen vermieden werden.
- **Priorität:** Niedrig (Technical Debt)

### 2.3 Abhängigkeits-Konflikte (`MSB3277`)
Es besteht ein Versionskonflikt bei `Microsoft.EntityFrameworkCore.Relational` (Version 10.0.1 vs 10.0.2). NuGet löst dies aktuell zugunsten der neueren Version auf, aber eine Konsolidierung der Pakete wird empfohlen.
- **Priorität:** Niedrig

### 2.4 Test-Qualität (`xUnit1012`)
In `PageServiceTests.cs` wird `null` für einen String-Parameter in einer `InlineData` verwendet, was eine xUnit-Warnung auslöst. Die Tests laufen zwar korrekt, der Code sollte aber auf `[Theory]`-kompatible Weise angepasst werden.

## 3. Funktionale Einschränkungen

- **Kein Media-Upload:** Das System unterstützt nativ keine Bilder-Uploads. Bilder müssen extern gehostet oder als Base64 eingebettet werden. (Dies ist eine bewusste Design-Entscheidung zur Reduktion der Angriffsfläche).
- **Linearer Verlauf:** Die Wiederherstellung einer alten Version erzeugt eine neue Version (Copy-Forward). Dies erhält die Historie, kann aber bei sehr vielen Änderungen die Datenbank vergrößern.

## 4. Fazit
Das System ist stabil und für den produktiven Einsatz geeignet, sofern die Deployment-Hinweise (Nginx-Konfiguration) beachtet werden. Die unter Punkt 2 genannten Warnungen sollten im nächsten Wartungszyklus behoben werden.
