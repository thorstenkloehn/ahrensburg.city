# Sicherheitsbericht "Status 24.03.2026" - MeinCMS

Dieser Bericht bewertet den aktuellen Sicherheitszustand der Software MeinCMS und gibt eine Empfehlung zur Veröffentlichung ab.

## 1. Aktueller Status der Software-Schwachstellen

### 1.1 Kritische Schwachstellen (behoben)
- **Passwort-Richtlinien:** Die Diskrepanz zwischen Web-App und `UserAdmin` wurde beseitigt. Beide erzwingen nun eine Mindestlänge von 12 Zeichen sowie Komplexitätsregeln.
- **Passwort-Exposition:** Im `UserAdmin`-Tool werden Passwörter bei der Eingabe nun durch Sternchen (`*`) maskiert.
- **Administrative Sicherheit:** Die automatische Löschung von "Zusatz-Admins" wurde durch ein Warnsystem ersetzt, um Datenverlust zu verhindern.

### 1.2 Verbleibende Schwachstellen (Kategorie: Gering bis Mittel)
- **Content Security Policy (CSP):** Die aktuelle CSP nutzt `'unsafe-inline'` für Skripte und Styles. 
    - **Risiko:** Erhöht das Risiko von Cross-Site Scripting (XSS), falls die HTML-Bereinigung (`HtmlSanitizer`) eine Lücke aufweisen sollte.
    - **Verbesserung:** Einsatz von Nonces (Einmal-Tokens) oder Hashes für Inline-Elemente.
- **Datei-Uploads (Untersagt):** Es wurde explizit festgelegt, dass das Hochladen von Dateien (Bilder, Dokumente) verboten ist.
    - **Status:** Die Funktionalität ist nicht vorhanden und darf aus Sicherheitsgründen nicht nachgerüstet werden.
    - **Sicherheitsvorteil:** Hierdurch wird der größte Angriffsvektor für Remote Code Execution (RCE) und Path Traversal vollständig eliminiert.
    - **Empfehlung:** Das Vorhaben sollte dauerhaft aus der Roadmap entfernt werden.

## 2. Bewertung der Veröffentlichungsfähigkeit

**Empfehlung: EINGESCHRÄNKT VERÖFFENTLICHUNGSFÄHIG**

Die Website kann technisch veröffentlicht werden, da die wesentlichen Schutzmechanismen (Anti-Forgery, HSTS, CSP-Grundgerüst, HTML-Sanitization, Account-Lockout) aktiv sind. Die öffentliche Registrierung ist sicher deaktiviert (`404 Not Found`).

**Was ist noch falsch / zu verbessern?**
1. **CSP Härtung:** Entfernen von `'unsafe-inline'` zur Erreichung eines "A"-Ratings in Sicherheits-Scans.
2. **Logging:** Eine zentrale Protokollierung von fehlgeschlagenen Admin-Anmeldeversuchen sollte implementiert werden.
3. **Validierung:** In `PageController` wurden bereits Prüfungen für Kategorien hinzugefügt (max. 10 Kategorien, max. 50 Zeichen), was die Stabilität verbessert.

## 3. Fazit
Die Software hat seit den letzten Audits massive Fortschritte gemacht. Die kritischen "Rich Risk"-Punkte im Admin-Tool wurden entschärft. Nach der Härtung der CSP ist das System bereit für den produktiven Einsatz.
