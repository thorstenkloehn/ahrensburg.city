## 2. Gefundene Schwachstellen & Bugs

---

### [KRITISCH] BUG-01 — Account-Lockout war deaktiviert

| | |
|---|---|
| **Datei** | `mvc/Areas/Identity/Pages/Account/Login.cshtml.cs:114` |
| **Status** | Behoben am 10.04.2026 |

**Problem:**  
Obwohl in `Program.cs` ein Account-Lockout konfiguriert war (5 Fehlversuche → 15 Minuten Sperre), wurde dieser nie ausgelöst. Der Aufruf `PasswordSignInAsync(..., lockoutOnFailure: false)` hat die gesamte Lockout-Konfiguration wirkungslos gemacht. Angreifer konnten unbegrenzt Passwörter ausprobieren (Brute-Force-Angriff).

**Behobene Zeile:**
```csharp
// Vorher (unsicher):
PasswordSignInAsync(Input.Email, Input.Password, Input.RememberMe, lockoutOnFailure: false);

// Nachher (korrekt):
PasswordSignInAsync(Input.Email, Input.Password, Input.RememberMe, lockoutOnFailure: true);
```

**Auswirkung bei Nichtbehebung:** Vollständige Umgehung des Brute-Force-Schutzes.

### [MITTEL] SCHWACH-01 — Schwaches Datenbankpasswort in Konfiguration

| | |
|---|---|
| **Datei** | `mvc/appsettings.json` |
| **Status** | Offen — muss vor Go-Live behoben werden |

**Problem:**  
Die produktive Konfigurationsdatei enthält `Password=Test` als Datenbankpasswort. Die Datei ist zwar über `.gitignore` aus dem Git-Repository ausgeschlossen, aber auf dem Server selbst ist dieses Passwort zu schwach.

**Empfehlung:**  
Starkes, zufällig generiertes Passwort setzen (min. 20 Zeichen, Sonderzeichen). Beispiel:
```bash
openssl rand -base64 32
```
Das Passwort dann in `appsettings.json` (Server) und in der PostgreSQL-Benutzerkonfiguration aktualisieren.

### [MITTEL] SCHWACH-02 — E-Mail-Versand nicht konfiguriert

| | |
|---|---|
| **Datei** | `mvc/Program.cs:55` |
| **Status** | Offen — muss vor Go-Live behoben werden |

**Problem:**  
`RequireConfirmedAccount = true` ist korrekt gesetzt, aber es gibt keinen echten E-Mail-Dienst (SMTP). Die folgenden Funktionen sind dadurch nicht einsatzbereit:
- Passwort zurücksetzen
- Neue Benutzer-Registrierung (Bestätigungsmail)
- E-Mail-Adresse ändern

**Empfehlung:**  
SMTP-Dienst einrichten, z. B. mit dem `MailKit`-Paket für ASP.NET Core Identity. Konfiguration über `appsettings.json`:
```json
"Email": {
  "SmtpServer": "mail.example.de",
  "SmtpPort": 587,
  "SenderEmail": "noreply@ahrensburg.city",
  "SenderName": "ahrensburg.city"
}
```

---

### [EMPFEHLUNG] EMP-01 — AllowedHosts einschränken

| | |
|---|---|
| **Datei** | `mvc/appsettings.json` |
| **Status** | Optional |

**Problem:**  
`"AllowedHosts": "*"` akzeptiert Anfragen von beliebigen Hostnamen. Das ist in Kombination mit dem Nginx-Reverse-Proxy unkritisch, aber nicht optimal.

**Empfehlung:**
```json
"AllowedHosts": "ahrensburg.city;doc.ahrensburg.city"
```

## 5. Installation auf Entwicklungsrechnern

### Voraussetzungen

| Software | Version | Download |
|---|---|---|
| .NET SDK | 10.0 | https://dotnet.microsoft.com/download |
| PostgreSQL | 15 oder höher | https://www.postgresql.org/download/ |
| Git | aktuell | https://git-scm.com |

### Schritt-für-Schritt

**1. Repository klonen:**
```bash
git clone <repository-url>
cd ahrensburg.city
```

**2. PostgreSQL-Datenbank anlegen:**
```sql
CREATE USER meincmsuser WITH PASSWORD 'IhrLokalPasswort';
CREATE DATABASE meincmsdb OWNER meincmsuser;
```

**3. Konfiguration erstellen (aus Vorlage):**
```bash
cp mvc/_appsettings.json mvc/appsettings.json
# appsettings.json anpassen: DB-Verbindung, Kestrel:UnixSocket auf "" setzen
```

**4. Datenbank migrieren und starten:**
```bash
dotnet run --project mvc -- --migrate
dotnet run --project mvc
```

**5. Ersten Admin-Benutzer anlegen:**
```bash
dotnet run --project UserAdmin
```

**6. Tests ausführen:**
```bash
dotnet test mvc.Tests
dotnet test Mardown.Tests
```

### Hinweise für Entwickler

- Die Datei `mvc/appsettings.json` wird von `.gitignore` ausgeschlossen — niemals committen.
- Nach Änderungen am Parser: `dotnet run --project backup -- repair` ausführen, um alle gespeicherten HTML-Inhalte neu zu generieren.
- Für Dokumentation: `docfx` (via `dotnet tools`).
