## Kein PHP auf meinem Server
Da der Code und die Bibliotheken nicht sicher überprüft werden können, ist PHP nicht sicher. Sicherheit kann nur gewährleistet werden, wenn der Code sicher ist und PHP regelmäßig aktualisiert wird. Ubuntu kann PHP-Updates schlecht handhaben. Python hingegen kann besser aktualisiert werden und ist leichter zu schreiben.
Ich benutze auch Node.js, Rust und Golang. Diese Programmiersprachen lassen sich gut aktualisieren. Java hingegen ist zu aufwändig - insbesondere Spring mit Java.
Einzelne Codes und Bibliotheken überprüfe ich mit folgenden Websites:

- [deps.dev](https://deps.dev)
- [snyk.io](https://snyk.io)
