# Meine Technologieauswahl

In diesem Dokument stelle ich meine bevorzugten Technologien in den Bereichen Headless CMS und Static Site Generators vor.

## Static Site Generator

Hier sind einige der Static Site Generators, die ich in Betracht ziehe:
## Die folgenden Static Site Generators sind nicht in Betracht zu ziehen:
### Leider kann ich nicht überprüfen, ob der Code Sicherheitslücken hat.
* **Jekyll**: Ein Ruby-basierter Static Site Generator. Ich kann nicht überprüfen, ob der Code Sicherheitslücken hat.
### Fehler beim Auflösen von Abhängigkeiten gefunden
* **Hugo**: Ein Go-basierter Static Site Generator. Es wurden Sicherheitslücken gefunden: Verwendung eines riskanten kryptografischen Algorithmus in github.com/aws/aws-sdk-go
GO-2022-0646 und Fehler beim Auflösen von Abhängigkeiten, die zu einem unvollständigen oder ungenauen Abhängigkeitsdiagramm führen könnten.
* **Gatsby**: Ein React-basierter Static Site Generator, der eine Vielzahl von Plugins und Themes bietet. Gatsby hat Sicherheitslücken: Verwendung eines riskanten kryptografischen Algorithmus in github.com/aws/aws-sdk-go
Axios Cross-Site Request Forgery Anfälligkeit und Path traversal in webpack-dev-middleware
## Die folgenden Static Site Generators sind in Betracht zu ziehen und sind auch sicher.
* **Astro.build** ist ein JavaScript-basierter Static Site Generator, der eine schnelle und performante Entwicklung ermöglicht.
### Webframeworks in React
* **Next.js**: Ein React-basiertes Framework, das sowohl statische Seiten als auch serverseitig gerenderte Seiten generieren kann.
#### Nur SSR, nicht für SSG-Bereiche
* **Remix**: Ein React-basiertes Framework, das serverseitiges Rendern unterstützt.

## Headless CMS
### Die folgenden Headless CMS sind nicht in Betracht zu ziehen:
* **WordPress**: Ein PHP-basiertes CMS, das eine Vielzahl von Plugins und Themes bietet. WordPress hat Sicherheitslücken: Cross-Site Scripting (XSS) in WordPress. Es kann auch Probleme mit der Aktualisierung und dem Upgrade von PHP geben. Leider unterstützt mein Server nur PostgreSQL und nicht MySQL.
* **Drupal**: Ein PHP-basiertes CMS, das eine Vielzahl von Plugins und Themes bietet. Drupal hat Sicherheitslücken: Cross-Site Scripting (XSS) in Drupal. Es kann auch Probleme mit der Aktualisierung und dem Upgrade von PHP geben. Leider unterstützt mein Server nur PostgreSQL und nicht MySQL.
* **Strapi**: Ein Node.js-basiertes CMS, das eine Vielzahl von Plugins und Themes bietet. Strapi hat Sicherheitslücken: Verweigerung des Dienstes beim Parsen einer Tar-Datei aufgrund fehlender Validierung der Ordneranzahl.
### Die folgenden Headless CMS sind in Betracht zu ziehen und sind auch sicher.

* **Keystone.js:** Ein Node.js-basiertes CMS, das eine einfache und flexible Konfiguration bietet.
* **Payload CMS** Ein Node.js-basiertes CMS, das eine einfache und flexible Konfiguration bietet.
* **Wagtail:** Ein Python-basiertes CMS, das eine einfache und flexible Konfiguration bietet.
## Betracht zu ziehende Frameworks
* **Spring Boot**: Dieses Framework ist sehr aufwändig und erfordert viel Code. Es gibt nicht viele gute deutsche Bücher darüber und es fühlt sich an, als müsste man eine Doktorarbeit schreiben.
* **ASP Net Core**: Dieses Framework ist sehr kompliziert und nicht gut für Anfänger geeignet. Es gibt nicht viele gute deutsche Bücher darüber und die Wartung ist auch nicht gut.
* **Rust**: Für Rust gibt es noch nicht so viele Bibliotheken und Frameworks. Die vorhandenen Bibliotheken sind noch nicht so stabil.
* **Express.js**: Ein Node.js-basiertes Framework, das eine einfache und flexible Konfiguration bietet.
* **Django**: Ein Python-basiertes Framework, das eine einfache und flexible Konfiguration bietet.
* **Echo**: Ein Go-basiertes Framework, das eine einfache und flexible Konfiguration bietet.
